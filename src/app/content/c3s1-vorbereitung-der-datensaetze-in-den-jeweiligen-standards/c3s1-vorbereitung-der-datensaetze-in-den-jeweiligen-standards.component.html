Die Datensätze müssen vor Ihrer Bereitstellung zuerst für den jeweiligen Standard aufgearbeitet werden. Dabei ist, je nach Standard, methodisch unterschiedlich vorzugehen. Ziel dieser Arbeit ist es, alle benötigten Schritte mithilfe von Open-source Software durchzuführen, um die einzelnen Arbeitsschritte für jeden Anwender auch ohne proprietäre Software zu ermöglichen.

Die Arbeitsschritte werden dabei nicht über eine Grafische Benutzeroberfläche (), wie sie durch QGIS bereitgestellt wird, ausgeführt. Diese Entscheidung wurde getroffen, um zum einen mögliche Eingabefehler zu vermeiden und die Ergebnisse reproduzierbar zu machen, und zum anderen die Möglichkeit einer Automatisierung der Vorbereitung der Daten zu bieten.

Als Werkzeug hierfür eignet sich besonders GDAL. Diese Bibliothek wird von der Open Source Geospatial Foundation () veröffentlicht und gepflegt und bietet dem Nutzer die gewünschte Möglichkeit, die Prozesse über das Terminal zu starten. Sofern QGIS auf dem jeweiligen Computer installiert ist, wird auch GDAL installiert, da QGIS selbst GDAL nutzt. Jedoch ist es hier wichtig, auf die installierte Version zu achten.

Mit Stand 30.01.2022 nutzt die aktuellste Version von QGIS (3.22.3 Białowieża vom 14.01.2022) nicht die aktuellste Version von GDAL, sondern die Version 3.0.4 vom 28.01.2020. Somit ist es nötig, sofern ebenfalls QGIS installiert ist, eine weitere Version von GDAL ergänzend zu installieren. Gleichzeitig muss jedoch die Version 3.0.4 für QGIS erhalten bleiben.

Der Optimized GeoTIFF- generator von GDAL wurde jedoch erst in der Version 3.1 veröffentlicht. Die Erstellung des COG ist durch die Version 3.0.4 folglich nicht möglich, daher ist es nötig, GDAL mithilfe von Anaconda, einer Software zum Arbeiten in Entwicklungsumgebungen, zu installieren. Anaconda ermöglicht die parallele Installation mehrerer Versionen der gleichen Software. Zudem lässt sich spezifisch die benötigte Version auswählen ohne eine bestehende Abhängigkeit, wie in diesem Fall von QGIS, zu gefährden. Diese Installationsmethode ist als "virtual environment" bekannt und bei Web-Entwicklern bzw. Data-Scientists, welche an verschiedenen Projekten arbeiten, weit verbreitet, da diese, wie in unserem Fall, meist verschiedene Versionen einer Software benötigen, um die unterschiedlichen Anforderungen der Projekte erfüllen zu können. So sollen die nachfolgenden Abschnitte die jeweils verschiedenen Methoden erklären.
