Die Bereitstellung des COG erfolgt vergleichbar zur Bereitstellung der Daten für den Standard TMS. Auch hier erfolgt dies, analog zu TMS, über einen HTTP-Server. Einziger Unterschied ist, dass der TMS aus mehreren Dateien besteht und das COG nur aus einem GeoTIFF. Folglich ist der Upload-Befehl zu AWS S3 ein anderer, da hier keine komplette Ordnerstruktur kopiert werden muss:

Zur Überprüfung, ob das COG korrekt aus AWS S3 abgelegt wurde, kann QGIS verwendet werden. Ein COG kann in QGIS über das Menü "Layer -> Add Layer -> Add Raster Layer ..." hinzugefügt werden. In dem Dialog wird anschließend von File auf Protocol: HTTP(S), cloud, etc. gewechselt und die URL des COG eingetragen. Das COG wird nun in QGIS als normaler Layer betrachtet und kann wie ein lokales File für Werkzeuge aus der Processing Toolbox verwendet werden.
