Nachdem das GeoTIFF übertragen wurde, muss es nun über den GeoServer veröffentlicht werden. Der Ablauf gliedert sich in mehrere Schritte: Erstellung des Arbeitsbereichs, Hinzufügen eines Datenspeichers, Veröffentlichung eines Layers und gegebenenfalls zusätzlich die Erstellung eines Stiels zur Visualisierung. Sämtliche Arbeitsschritte erfolgen nach Anmeldung auf der WebOberfläche des GeoServers.

Die Veröffentlichung eines Layers ohne einen festgelegten Arbeitsbereich ist nicht möglich. Dieser kann als Sammelobjekt für mehrere Layer verstanden werden. Für den vorliegenden Standard wurde ein Arbeitsbereich mit den Namen WMS erstellt.

Weitere Voraussetzung für einen Layer bildet der Datenspeicher als Datenquelle im Kontext des GeoServers für einen Layer. Der GeoServer bietet als Datenquellen Vektordaten, Rasterdaten und "andere Datenquellen". Im Rahmen dieser Arbeit geht es vorwiegend um die Veröffentlichung von Rasterdaten, daher werden die beiden anderen Kategorien nicht näher betrachtet. Als Datenquelle für Rasterdaten kann ArcGrid, GeoPackage, GeoTIFF ImageMosaic oder WorldImage genutzt werden. Die in dieser Arbeit verwendete Datenquelle aus diesen Kategorien ist das GeoTIFF. Im Interface muss der Arbeitsbereich gewählt werden, ein Name der Datenquelle festgelegt und der Verbindungsparameter, welcher den Pfad zu dem GeoTIFF darstellt. Dieser muss nicht manuell ausgewählt werden, sondern kann über das "Browse..." komfortabel per GUI selektiert werden. Die nachfolgende Abbildung stellt die Eingaben beispielhaft dar:

Nachdem nun der Datenspeicher hinzugefügt wurde, kann in der Ansicht "Neuer Layer" dieser ohne Verzögerung publiziert werden. Die Publikation öffnet direkt das Interface zum Bearbeiten des Layers, in dem sich mehrere Reiter zur weiteren Bearbeitung finden: Daten, Publizierung, Ausdehnung, Kartenkachel-Cache und Security. In Daten muss ein Name für den Layer vergeben sowie das Koordinatensystem und die Bänderdetails geprüft werden. In dem Tab Publizierung können der Autor der Daten bzw. des WMS genannt werden und dessen URL zur Homepage. Ziel dieses Kapitels ist, es einen WMS zu veröffentlichen, somit muss in dem Tab Kartenkachel-Cache die Checkbox "Erzeuge einen gecachten Layer für diesen Layer" deaktiviert werden. In den Tabs Ausdehnung und Security werden keine Änderungen vorgenommen. Nach allen erforderlichen Eingaben kann der Layer nun gespeichert werden.

Mithilfe der Layer-Vorschau kann der erzeugte Layer in einer OpenLayers auf Richtigkeit überprüft werden. Diese Vorschau ist auch für Nutzer, welche über keinen Login zum GeoServer verfügen, möglich. Hierfür muss nur die Domain des GeoServers geoserver.masterarbeit-cog.de/web/ aufgerufen und im linksseitigen Menü die Layer-Vorschau ausgewählt werden. Als Dateiformat ist hier "Styled Layer Descriptor" () zu wählen, da dies ebenfalls vom GeoServer unterstützt wird.

In der WebOberfläche des GeoServers ist im nächsten Schritt auf der linken Seite der Eintrag "Stile" zu wählen, welcher dem "Style" des QGIS entspricht. In diesem wird anschließend ein neuer hinzugefügt. In der Eingabemaske wird nun ein Name für den Stil eingetragen, das Format SLD gewählt und der Inhalt aus der zuvor gespeicherten Datei eingefügt. Diese Eingabe sollte dann der folgenden Abbildung entsprechen. Um diesen Stil zu speichern ohne die Eingabemaske zu verlassen, wird muss "Apply" gewählt.

Ist die Speicherung erfolgt, muss der Stil noch dem passendem Layer hinzugefügt werden. Dies erfolgt im Tab "Publishing", in welchem eine Selektion der gewünschten Layer vorgenommen werden kann.
